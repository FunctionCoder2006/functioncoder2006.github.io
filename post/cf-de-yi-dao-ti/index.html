这道题是这样的：给定一个数列 aia_iai​ , 让你任意改变数列中元素的顺序，求 ∑i=1ngcd(a1,a2,a3……ai)\sum_{i = 1}^{n} gcd(a_1 , a_2 , a_3……a_i)∑i=1n​gcd(a1​,a2​,a3​……ai​) 最大

如果真的要考虑每个元素的顺序的话，那只能状压，但是这道题最大元素 ≤5e6≤ 5e6≤5e6 ，因此考虑将值记入状态里面

设 fif_ifi​ 表示当前生成的序列 gcdgcdgcd 为 iii 能得到的最大 gcdgcdgcd 和

考虑 fif_ifi​ 由谁转移过来（如何划分集合），我们不难发现，随着序列的加长 gcdgcdgcd 会不断减小并且会减小到 tatata 的约数，因此，有

fi=max(fj+sumi,j),j∣if_{i} = max(f_j + sum_{i , j}) , j | i fi​=max(fj​+sumi,j​),j∣i

因为当前生成的序列的 gcdgcdgcd 为 iii , tatata 的某一个前缀 gcdgcdgcd 为 jjj，所以会形成一段贡献为 iii 的区间，故 sumi,jsum_{i , j}sumi,j​ 表示这个区间的贡献

[[jjjjjjjjj]iiiiiiiiii][[jjjjjjjjj]iiiiiiiiii][[jjjjjjjjj]iiiiiiiiii]

然而我们对 sumi,jsum_{i , j}sumi,j​ 束手无策，考虑到，如果我们将原序列的一个元素 iii 放到了 iii 的约数那一段也就是说我们将 iii 后放了导致贡献减小了，因为往后面放的话，你再大也会被拽下来

所以，我们将所有不是 jjj 的倍数并且是 iii 的倍数全都放到这个区间里面产生 iii 的贡献，故有

sumi,j=i∗(cnti−cntj)sum_{i , j} = i * (cnt_i - cnt_j) sumi,j​=i∗(cnti​−cntj​)

其中，cnticnt_icnti​ 表示原数列中是 iii 的倍数有多少个

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int N = 1e5 + 10;
const int M = 5e6 + 10;

int n , a[N] , f[M] , cnt[M] , maxn , ans;
bool st[M];

int main(void){

    scanf("%d" , &n);

    for(int i = 1 ; i <= n ; ++ i){
        scanf("%d" , &a[i]);
        maxn = max(maxn , a[i]);
        st[a[i]] = true;
    }

    for(int i = 1 ; i <= n ; ++ i){
        for(int j = i ; j <= n ; j += i){
            if(st[j]){
                cnt[i] ++ ;
            }
        }
    }

    for(int i = maxn ; i >= 1 ; -- i){
        for(int j = i * 2 ; j <= maxn ; j += i){
            f[i] = max(f[i] , f[j] + (cnt[i] - cnt[j]) * i);
        }
        ans = max(ans , f[i]);
    }

    printf("%d" , ans);

    return 0;

}
