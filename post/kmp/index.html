2021-10-02 #KMP #字符串

kmp一共有两个步骤，一个是自己和自己匹配，一个是自己和文本串匹配

首先是自己和自己匹配：
nxinx_inxi​表示模式串以i为后缀（非前缀）和模式串的前缀的最大匹配长度。假设nx1 nxi−1nx_1 ~ nx_{i - 1}nx1​ nxi−1​已经算出。假如枚举到某一个iii发现已经失配，但是我又不想从头开始配对，我又想尽可能让iii向前延拓从而让nxinx_inxi​是最大值

（红的a表示当前iii的位置）这里a已经不等于b了即失配了，所以我要想办法尽可能让向前延拓。但是注意此时已经不可能延拓超过5个字符因为nxi−1=5nx_{i - 1} = 5nxi−1​=5而nxnxnx值又是最大的即我在i−1i - 1i−1的时候最多能延拓5个字符，而匹配到第iii位的时候又失配了所以我是不可能延拓超过5个字符的。那，能不能等于5个字符呢？不能！因为虽然nxi−1=5nx_{i - 1} = 5nxi−1​=5即在i−1i - 1i−1能延拓5个字符，但是就算这样，也肯定会失败的因为我就是nxnxnx值达到5的时候失配了，延拓5个相当于再撞一次墙。综上，不可能延拓超过5个字符。那么我最多能延拓多少呢？当然就是nxjnx_jnxj​了。因为我想要尽可能向前延拓又想让以iii为后缀和模式串前缀相等，根据nxnxnx定义，自然就是nxjnx_jnxj​

其次是自己和文本串匹配：
还是同样的道理，当发生失配时，要想办法尽可能向前延拓，而因为i−1i - 1i−1之前都已经配对成功了，所以以i−1i - 1i−1为后缀长度为nxi−1nx_{i - 1}nxi−1​的串自然就是模式串的前缀。读到这里，应该能联想到刚刚是怎么处理nxnxnx数组了，也是同样的道理，发生失配时跳nxnxnx就可以了。这里注意当匹配成功时可能还有下一次成功匹配，所以要先令j=nxjj = nx_jj=nxj​。为什么呢？其实也是同样的道理，尽可能向前延拓


#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cmath>
#include<cstring>

using namespace std;

const int N = 1e6 + 10;

int n , m , nx[N];
char a[N] , b[N];

int main(void){

    scanf("%s" , a + 1);
    scanf("%s" , b + 1);

    n = strlen(a + 1) , m = strlen(b + 1);

    nx[1] = 0;
    int j = 0;
    for(int i = 2 ; i <= m ; ++i){
        while(j > 0 && b[i] != b[j + 1])
            j = nx[j];
        if(b[i] == b[j + 1])
            ++ j;
        nx[i] = j;
    }
    j = 0;
    for(int i = 1 ; i <= n ; ++i){
        while(j > 0 && a[i] != b[j + 1])
            j = nx[j];
        if(a[i] == b[j + 1])
            ++ j;
        if(j == m){
            printf("%d\n" , i - m + 1);
            j = nx[j];
        }
    }
    for(int i = 1 ; i <= m ; ++i){
        printf("%d " , nx[i]);
    }

    return 0;

}
