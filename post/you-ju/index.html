邮局

2021-12-07 #线性dp #动态规划

这道题嗑了好长时间，终于拿到了 40pts40pts40pts

设 fi,jf_{i , j}fi,j​ 表示，所有只考虑前 iii 个村庄，已经设置了 jjj 和邮局的方案中的最小值是多少

但是我们很快发现没有办法转移。邮局设在哪里？某一个村庄该往那个邮局跑？

我们考虑给状态添加含义，令 iii 设置邮局。那么我们考虑上一个邮局在哪里设置的，我们设为 kkk。我们将 iii 和 jjj 中间的距离等分，等分线左边的去 kkk 邮局，右边的去 iii 邮局，距离的话只需要使用前缀和记录 cntcntcnt 和 sumsumsum 就可以了

我们考虑最后一个邮局是设在哪里的，因此，我们只需要打擂台求出最大的 fi,mf_{i , m}fi,m​ 就可以了，因为 iii 表示只考虑前 iii 个村庄并且 iii 设置了村庄，那最后一个村庄显然应该是 iii

但是我们会漏掉一个点，右面的村庄也需要到左面的邮局，我们还需要将其距离加上

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int N = 550;
const int M = 1e4 + 10;

int n , m , p[N] , sum[M] , cnt[M] , maxd;
int f[N][N];

int main(void){

    scanf("%d%d" , &n , &m);

    for(int i = 1 ; i <= n ; ++ i){
        scanf("%d" , &p[i]);
        maxd = max(maxd , p[i]);
    }

    sort(p + 1 , p + 1 + n);

    for(int i = 1 ; i <= n ; ++ i){
        sum[p[i]] += p[i];
        cnt[p[i]] ++ ;
    }

    for(int i = 1 ; i <= maxd ; ++ i){
        sum[i] += sum[i - 1];
        cnt[i] += cnt[i - 1];
    }

    memset(f , 0x3f , sizeof(f));

    for(int i = 1 ; i <= n ; ++ i){
        f[i][1] = cnt[p[i]] * p[i] - sum[p[i]];
    }

    for(int j = 2 ; j <= m ; ++ j){
        for(int i = 2 ; i <= n ; ++ i){
            for(int k = 1 ; k < i ; ++ k){
                int mid = (p[i] + p[k]) >> 1;
                int c = sum[mid] - sum[p[k] - 1] - (cnt[mid] - cnt[p[k] - 1]) * p[k];
                c += p[i] * (cnt[p[i]] - cnt[mid]) - (sum[p[i]] - sum[mid]);
                f[i][j] = min(f[i][j] , f[k][j - 1] + c);
            }
        }
    }

    int ans = 2e9;

    for(int i = 1 ; i <= n ; ++ i){
        ans = min(ans , f[i][m] + sum[maxd] - sum[p[i]] - p[i] * (cnt[maxd] - cnt[p[i]]));
    }
    
    printf("%d" , ans);

    return 0;

}

还有一点，在给前缀和数组赋值的时候我将 nnn 写成了 maxdmaxdmaxd 导致了前缀和数组偏大，以后一定要注意

后来又想了想中位数的做法，那个做法没有要求 iii 设置邮局，并且将最后一个区间 [k+1,i][k + 1 , i][k+1,i] 的邮局设在了中位数里。之前一直对这个做法感到疑惑，万一 [k+1,i][k + 1 , i][k+1,i] 存在区间里的某个村庄，离这个村庄的最近的邮局不是这个设在中位数的邮局，那不就错了吗？

是这样的，之前出现好多遍了，因为我在划分区间的时候，一定会枚举到更优的断点 kkk ，而这个最优的断点一定比计算出来错误的断点 kkk 要优，所以是不必担心的
