快递服务题解

2021-10-17 #线性dp #动态规划

这道题又做了一遍，感觉之前写的题解不太妥当，于是有了这篇题解

首先很容易想到是设 fi,j,k,lf_{i , j , k , l}fi,j,k,l​ 表示所有已经执行了 iii 个指令并且 111 222 333 号员工分别恰好停到了 iii jjj kkk

注意我写的是“恰好”，就意味着可能有非法状态会造成干扰

如果要转移地话，我们只需要考虑哪一个员工到这个地方就行了

然后考虑优化

我们发现，如果已经执行了 iii 个指令，那么必定会有一个员工停在 aia_iai​ 点上，我们考虑剪去无用状态，因为剩下的状态肯定是非法的， 白白浪费时间枚举

这样我们就可以压掉一维，给 fff 数组添加含义，设 fi,j,kf_{i , j , k}fi,j,k​ 表示所有已经执行了 iii 个指令，并且三个员工停到了 jjj kkk aia_iai​

但是我们需要思考一个问题，我们需不需要考虑哪一个员工到这个地方？

其实是不需要的，我只需要知道三个员工在这三个位置就行了，不需要管这三个位置分别是哪一个员工，因为我转移的时候加上的是这个位置到下一个位置的距离，与员工的编号无关

但是我们初始化怎么办？f0,1,2f_{0 , 1 , 2}f0,1,2​ 和 f0,2,1f_{0 , 2 , 1}f0,2,1​ 不是等效的吗？

这时候我们直接令 f0,1,2=0f_{0 , 1 , 2} = 0f0,1,2​=0，将 f0,2,1f_{0 , 2 , 1}f0,2,1​ 设置成非法状态。因为既然他们两个是等效的，那么我为什么不只转移一个呢，两个都转移其实没有太大区别反而还会麻烦一些

转移的时候也是一样的，我既可以转移到 fi+1,k,jf_{i + 1 , k , j}fi+1,k,j​，也可以转移到 fi+1,j,kf_{i + 1 , j , k}fi+1,j,k​ ，这时候我们是不是舍弃一种就可以了

这里还需要注意一个细节，就是 j==kj == kj==k || ai==ja_i == jai​==j || ai==ka_i == kai​==k 的时候，此时我的状态是非法的，虽然我已经设置了 0x3f0x3f0x3f 来剪去非法状态，但是我仍然要特判这种情况。因为我的这个状态有可能是从合法状态转移过来的，因此此时这个非法状态是有合法值的，并且这个非法状态还会去影响其他合法状态，导致答案错误

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int M = 2e2 + 10;
const int N = 2e3 + 10;

int m , n , a[N] , f[2][M][M] , d[M][M];

int main(void){
    
    ios::sync_with_stdio(false); cin.tie(0); cout.tie(0);
    
    cin >> m >> n;
    
    for(int i = 1 ; i <= m ; ++ i){
        for(int j = 1 ; j <= m ; ++ j){
            cin >> d[i][j];
        }
    }
    
    for(int i = 1 ; i <= n ; ++ i){
        cin >> a[i];
    }
    
    memset(f , 0x3f , sizeof(f));
    
    f[0][1][2] = 0 , a[0] = 3;
    
    for(int i = 0 ; i <= n ; ++ i){
        memset(f[(i + 1) & 1] , 0x3f , sizeof(f[(i + 1) & 1]));
        for(int j = 1 ; j <= m ; ++ j){
            for(int k = 1 ; k <= m ; ++ k){
                if(j == k || a[i] == k || a[i] == j)continue;//
                f[(i + 1) & 1][j][k] = min(f[(i + 1) & 1][j][k] , f[i & 1][j][k] + d[a[i]][a[i + 1]]);
                f[(i + 1) & 1][j][a[i]] = min(f[(i + 1) & 1][j][a[i]] , f[i & 1][j][k] + d[k][a[i + 1]]);
                f[(i + 1) & 1][k][a[i]] = min(f[(i + 1) & 1][k][a[i]] , f[i & 1][j][k] + d[j][a[i + 1]]);
            }
        }
    }
    
    int ans = (1 << 30);
    
    for(int j = 1 ; j <= m ; ++ j){
        for(int k = 1 ; k <= m ; ++ k){
            ans = min(ans , f[n & 1][j][k]);
        }
    }
    
    cout << ans << endl;
    
    return 0;
    
}

最核心的，我们需要掌握 dpdpdp 的一个优化技巧，剪去无用状态，尝试着压掉一维度
