CQOI涂色

Posted on 2021-12-26，4 min read

当我们读完题我们可以大致感受一下，肯定是先涂大区间然后再涂它背后的小区间

在涂抹一个区间的时候，我们有两种策略

一种是直接给大区间涂，给大区间直接涂又分成两种情况

  • 一种是 str[l]≠str[r]str[l] ≠ str[r]str[l]​=str[r] ，也就是左右端点不相等，因为要先涂大区间所以我们直接让整个区间涂成左端点颜色或者右端点颜色

  • 另一种是 str[l]=str[r]str[l] = str[r]str[l]=str[r] 我们直接让整个大区间涂成左右端点的颜色

综上，当 str[l]≠str[r]str[l] ≠ str[r]str[l]​=str[r] 的时候，状态可以转移到 fl,r−1+1f_{l , r - 1} + 1fl,r−1​+1 或者 fl+1,r+1f_{l + 1 , r} + 1fl+1,r​+1 这里用到了一个等效手法，注意等效的是涂了以后的结果而不是状态；当 str[l]=str[r]str[l] = str[r]str[l]=str[r] 的时候，状态可以转移到 fl+1,rf_{l + 1 , r}fl+1,r​ 或者 fl,r+1f_{l , r + 1}fl,r+1​ 这里涂了的结果的等效状态我没有想到

另一种是把大区间分成两个小区间涂，有

fl,r=max(fl,r,fl,k+fk+1,r)，k∈[l,r−1]f_{l , r} = max(f_{l , r} , f_{l , k} + f_{k + 1 , r})，k ∈[l , r - 1] fl,r​=max(fl,r​,fl,k​+fk+1,r​)，k∈[l,r−1]

非常神奇的是这种区间划分的情况包含了上面的 str[l]≠str[r]str[l] ≠ str[r]str[l]​=str[r] 的情况。当 k=r−1k = r - 1k=r−1 时 fl,r=max(fl,r,fl,r−1+fr−1+1,r)f_{l , r} = max(f_{l , r} , f_{l , r - 1} + f_{r - 1 + 1 , r})fl,r​=max(fl,r​,fl,r−1​+fr−1+1,r​) 因为 fr,r=1f_{r , r} = 1fr,r​=1，因此 fl,r=max(fl,r,fl,r−1+1)f_{l , r} = max(f_{l , r} , f_{l , r - 1} + 1)fl,r​=max(fl,r​,fl,r−1​+1)。

k=lk = lk=l 也是一样的道理

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int N = 110;

char str[N];
int f[N][N];

int main(void){

    scanf("%s" , str + 1);

    int n = strlen(str + 1);

    memset(f , 0x3f , sizeof(f));

    for(int i = 1 ; i <= n ; ++ i){
        f[i][i] = 1;
    }

    for(int Len = 2 ; Len <= n ; ++ Len){
        for(int l = 1 ; l + Len - 1 <= n ; ++ l){
            int r = l + Len - 1;
            if(str[l] == str[r]){
                f[l][r] = min(f[l + 1][r] , f[l][r - 1]);
            }
            for(int k = l ; k < r ; ++ k){
                f[l][r] = min(f[l][r] , f[l][k] + f[k + 1][r]);
            }
            f[l][r] = min(f[l][r - 1] + 1 , f[l + 1][r] + 1);
        }
    }

    printf("%d" , f[1][n]);

    return 0;

}
