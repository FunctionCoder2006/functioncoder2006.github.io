宠物小精灵之收服题解

2021-11-11 3 min read # 背包 # 动态规划

这道题本身不难，就是一个细节需要注意：皮卡丘的体力不能为 0 ，抽象成背包语言就是背包不能恰好装满。

发现这个问题以后，一开始把 r>=c[i]r >= c[i]r>=c[i] 改成了 r>=c[i]+1r >= c[i] + 1r>=c[i]+1 但是想了想，这么做是不对的。如果假设我装了两个物品，一个重量为 w1w_1w1​，另一个重量为 w2w_2w2​ ，且保证 w1+w2w_1 + w_2w1​+w2​ < mmm，我是完全可以把两个物品全都装下的，但是执行到程序中 fw1+w2f_{w_1 + w_2}fw1​+w2​​就会显示只能装一个，因为 fw1f_{w_1}fw1​​ 是 0，fw1+w1=fw1+w2−w2+1=fw1+1=1f_{w1 + w1} = f_{w_1 + w_2 - w_2} + 1 = f_{w_1} + 1 = 1fw1+w1​=fw1​+w2​−w2​​+1=fw1​​+1=1

我的解决方案是使用一个辅助数组 gjg_jgj​ ，表示在最优决策中，是否是恰好装了体积为 jjj 的物品，因为 dpdpdp 数组的 jjj 表示的并不是恰好装了体积为 jjj 的物品，而是装的物品不超过 jjj，具体实现可以看代码。

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int N = 1e3 + 10;
int n , k , m , c[N] , w[N] , f[N][N] , g[N];

int main(void){

    scanf("%d%d%d" , &k , &m , &n);

    for(int i = 1 ; i <= n ; ++ i){
        scanf("%d%d" , &w[i] , &c[i]);
    }

    g[0] = true;    

    for(int i = 1 ; i <= n ; ++ i){
        for(int l = k ; l >= w[i] ; -- l){
            for(int r = m ; r >= c[i] ; -- r){
                if(f[l - w[i]][r - c[i]] + 1 > f[l][r]){
                    f[l][r] = f[l - w[i]][r - c[i]] + 1; 
                    g[r] = g[r - c[i]];
                }
            }
        }
    }

    int ans = 0 , youwike = 0;

    for(int r = 0 ; r <= m ; ++ r){
        if(r == m && g[r]) -- f[k][r];
        ans = max(ans , f[k][r]);
    }

    for(int r = 0 ; r <= m ; ++ r){
        if(f[k][r] == ans){
            youwike = r;
            break;
        }
    }

    printf("%d %d" , ans , m - youwike);

    return 0;

}
