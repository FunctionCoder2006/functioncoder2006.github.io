考虑暴力如何做。因为题目中要求前后缀不能重叠，所以对于每一个iii，如果发现他的最长border是重叠的，就让它跳nxnxnx直到不能跳为止，途中如果发现border长度符合题设，那么numi++num_i ++numi​++。因为一直跳nxnxnx可以遍历到ta的所有border。

但是如果遇到aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa这种数据的话容易被卡成O(N2)O(N^2)O(N2)因为会跳nxnxnx很多次。所以我们考虑对跳跃加速，于是我们想到了倍增（虽然当时我没有想到）。

重头戏是如何倍增。回忆倍增求LCA，我们将depx−depsdep_x - dep_sdepx​−deps​拆成若干个2的幂次和，然后先跳幂次方大的。那么这道题也是同样的道理，先尝试着尽量往远处跳但是不能跳到border小于i/2i / 2i/2的地方，最后再向跳一下，这样就是满足条件的最长border了。

记录cnticnt_icnti​表示以iii结尾的子串的border个数（先不考虑重复）提前预处理cntcntcnt。cnti=cntj+1cnt_i = cnt_j + 1cnti​=cntj​+1其中jjj是ta的最长border（因为jjj一定小于iii所以可以递推）。

注意cntcntcnt最初要设成1因为考虑cntcntcnt的定义，此时我们暂且不考虑重复，不考虑重复的话一定会把前缀是ta后缀也是ta的情况算上。

当我们跳到最长满足要求的border时，就可以让ta的cntcntcnt算上贡献了，因为cntcntcnt里有着ta本身的一种情况，cntcntcnt又记录着ta的border们的前缀和，所以是符合题设的。

#include<iostream>
#include<cstdio>
#include<algorithm>
#include<cstring>
#include<cmath>

using namespace std;

const int N = 1e6 + 10;
const int M = 21;
const int mod = 1e9 + 7;

int t;
char str[N];
int nx[N] , p[M][N] , cnt[N] , Log[N];

int main(void){

    scanf("%d" , &t);

    Log[1] = 0;
    for(int i = 2 ; i < N ; ++i){
        Log[i] = Log[i >> 1] + 1;
    }

    while(t -- ){
        scanf("%s" , str + 1);
        int n = strlen(str + 1);
        memset(nx , 0 , sizeof(nx));
        memset(cnt , 0 , sizeof(cnt));
        memset(p , -1 , sizeof(p));
        int j = 0; nx[1] = 0; cnt[1] = 1;
        for(int i = 2 ; i <= n ; ++i){
            while(j > 0 && str[i] != str[j + 1])
                j = nx[j];
            if(str[i] == str[j + 1])
                ++ j;
            nx[i] = j;
            p[0][i] = j , cnt[i] = cnt[j] + 1;
        }
        for(int j = 1 ; (1 << j) <= n ; ++j){
            for(int i = 1 ; i <= n ; ++i){
                if(p[j - 1][i] != -1)
                    p[j][i] = p[j - 1][p[j - 1][i]];
            }
        }
        long long res = 1ll;
        for(int i = 1 ; i <= n ; ++i){
            int x = i;
            for(int j = Log[cnt[n]] ; j >= 0 ; --j){
                if(p[j][x] != -1 && p[j][x] * 2 > i)
                    x = p[j][x];
            }
            x = p[0][x];
            res = 1ll * res * (cnt[x] + 1) % mod;
        }
        printf("%lld\n" , res);
    }

    return 0;

}

正解是线性做法。

还是维护一个cntcntcnt数组，先不考虑重复计算前缀和，这样就不用接着往后跳了。

然后我们考虑如何快速计算numnumnum数组。我们只需要再进行一次类似KMP的流程，每次尝试着找到满足要求的最长border。如果发现不匹配，那就往回跳nxnxnx，找到（先不管重叠）最长的border。

当我们发现jjj（这里的jjj表示满足要求的最长border）已经大于 i2\frac{i}{2}2i​了说明已经不满足题设了，我们就令j=nxjj = nx_jj=nxj​，直到jjj满足条件。此时的jjj是满足题设的最长border，满足jjj的定义。

因为我每次都让jjj保持满足题设最大（类比KMP，因为KMP不就是保持jjj满足题设即前后缀匹配最大嘛），所以复杂度是O(N)O(N)O(N)的
